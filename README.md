# Civ Browser (MVP)

Хобби-проект: упрощённый браузерный клон Civilization 5. На этом этапе реализован
только **World Builder** — гекс-карта, на которой можно вручную расставлять
ландшафт, ресурсы и города. Игровая логика (ходы, юниты, бой, ИИ) добавляется
дальше поверх этого фундамента.

## Стек

- TypeScript + Vite + React — UI и сборка
- Canvas 2D API — рендер гекс-карты (без игрового движка, чтобы всё оставалось
  прозрачным и легко расширяемым)
- Zustand — состояние игры

## Как запустить

1. Установи зависимости:
   ```bash
   npm install
   ```
2. Запусти dev-сервер:
   ```bash
   npm run dev
   ```
3. Открой адрес, который покажет Vite (обычно http://localhost:5173).

## Как открыть в Cursor

1. Открой папку `civ-browser` в Cursor (`File → Open Folder`).
2. Открой встроенный терминал (`` Ctrl+` ``) и выполни команды из раздела выше.
3. Дальше можно просить Cursor/Claude в чате дополнять функционал — структура
   проекта уже настроена так, чтобы новый код ложился в понятные модули (см. ниже).

## Структура проекта

```
src/
  game/
    types.ts     — все игровые типы (Tile, City, Unit, Civilization, GameState)
    hexGrid.ts    — гекс-математика (координаты, соседи, расстояния, пиксели)
    store.ts      — Zustand-стор с состоянием игры и World Builder
  components/
    MapCanvas.tsx — отрисовка карты и обработка кликов
    Toolbar.tsx   — панель выбора ландшафта/ресурса/режима
  App.tsx
  main.tsx
```

## Текущий функционал

- Гекс-карта 220×160 (≈35 000 тайлов) — размер задаётся константами `MAP_WIDTH`/`MAP_HEIGHT` в `src/game/store.ts`
- Процедурная генерация базового континента/океана (`src/game/mapGenerator.ts`) — при загрузке и по кнопке "Сгенерировать заново" в тулбаре
- Камера: перетаскивание мышью (пан) и колесо мыши (зум) — реализовано отсечение невидимых тайлов, так что рендер не деградирует с ростом карты
- World Builder: выбор ландшафта / ресурса / режима "город" + **кисть с радиусом** (1 гекс / радиус 1-8) для быстрой ручной правки больших областей
- Клик без перетаскивания красит гекс(ы); перетаскивание — просто пан по карте

### Про размер карты

Дефолт — 220×160, это примерно в 5 раз больше, чем самая большая карта в
оригинальном Civ5 (Huge, 104×64). Дальше увеличивать можно, меняя константы в
`store.ts`, но помни: на масштабах в 100 000+ гексов начинают тормозить не
рендер (он уже с отсечением), а более "тяжёлые" вещи, которые будешь добавлять
позже — pathfinding юнитов и просчёт ходов ИИ по всей карте. Проверяй
производительность на своей машине при увеличении.

## Следующие шаги (предлагаемый порядок)

1. **Основание городов по-настоящему** — City как сущность с населением, а не просто метка на гексе
2. **Юниты и движение** — добавить Unit-сущности, стоимость перемещения по ландшафту (`hexGrid.ts` уже даёт `hexDistance` и `neighbors`)
3. **Ходы** — счётчик хода, кнопка "Закончить ход", обработка юнитов ИИ
4. **Производство в городах** — очередь построек/юнитов, простая формула стоимости
5. **Технологии** — минимальное дерево, очки науки от городов
6. **ИИ-логика** — алгоритмические правила для базовых решений; точки, где стоит звать LLM (Claude API), обозначить отдельно как "решение", а не встраивать вглубь кода
7. **Сохранение/загрузка** — экспорт `GameState` в JSON, импорт обратно
8. **Деплой** — когда будешь готов вынести в сеть: `npm run build` даёт статические файлы в `dist/`, которые можно залить на Vercel/Netlify и повесить домен — бэкенд для одиночной игры не нужен

## Замечания по архитектуре

- Все координаты — axial (`{ q, r }`), см. https://www.redblobgames.com/grids/hexagons/
- Карта хранится как `Record<string, Tile>` с ключом `"q,r"` — удобно для точечных обновлений без пересчёта всего массива
- Namespace-разделение уже заложено: игровая логика (`game/`) отдельно от UI (`components/`) — это пригодится, когда логику придётся выносить на сервер для мультиплеера
